# 🌱 Система автоматического полива "Томатный агрегат"

[![ESP-IDF](https://img.shields.io/badge/ESP--IDF-v6.1-blue)](https://github.com/espressif/esp-idf)
[![Platform](https://img.shields.io/badge/platform-ESP32--C3%20Super%20Mini-green)](https://www.espressif.com/en/products/socs/esp32-c3)
[![Language](https://img.shields.io/badge/language-C-00599C)](https://en.wikipedia.org/wiki/C_(programming_language))

Форк проекта [arregation_system_tomato](https://github.com/Raven078/arregation_system_tomato.git), значительно расширенный функционалом:

- Синхронизация времени по TCP‑команде `GET_TIMESTAMP` с удалённого сервера (ESP32‑S3 Touch LCD‑7).
- Управление насосом (ШИМ) и клапаном через драйвер L298N.
- Автоматическая логика полива: включение насоса при низкой влажности, времени до 11:00 и наличии воды на датчике уровня 1; выключение при пропадании уровня 1; открытие клапана через 10 секунд после остановки насоса и закрытие по датчику уровня 2.
- Логирование в энергонезависимую flash‑память (SPIFFS):
  - Файл с именем `DD_MM_YYYY.txt`.
  - Каждые 10 минут – запись усреднённых за 10 минут влажности и температуры (текущие уровни – мгновенные).
  - При событиях (включение/выключение насоса, открытие/закрытие клапана) – немедленная запись в файл.
  - При смене дня старый файл отправляется на сервер, удаляется, создаётся новый.
- Отправка данных на TCP‑сервер:
  - Каждую минуту – JSON с усреднёнными за минуту влажностью и температурой (текущие уровни и состояния насоса/клапана – мгновенные).
  - Немедленно при изменении состояний уровней, насоса или клапана – мгновенный JSON.
  - После каждого обновления (10‑минутная запись) файл лога отправляется на сервер.
- Отображение содержимого лог‑файла в мониторе после каждой отправки (для отладки).
- Калибровка датчика влажности через `menuconfig` (сухое и влажное напряжение в мВ).

## 🔌 Схема подключения (ESP32‑C3 Super Mini)

---------------------------------------------------------------------------------------------
| Компонент                 | Пин ESP32‑C3      | Примечание                                |
|---------------------------|-------------------|-------------------------------------------|
| **L298N (насос)**         | GPIO6 (ENA)       | ШИМ, управление скоростью                 |
|                           | GPIO7 (IN1)       | Включение (HIGH – вкл)                    |
|                           | GND (IN2)         | Заземлить                                 |
| **L298N (клапан)**        | GPIO8 (IN3)       | Включение (HIGH – вкл)                    |
|                           | GND (IN4)         | Заземлить                                 |
|                           | ENB               | Установить перемычку (ШИМ не используется)|
| **HW‑390** (влажность)    | GPIO1 (ADC1_CH0)  | Аналоговый выход                          |
| **DS18B20** (температура) | GPIO0             | Подтяжка 4.7 кОм к 3.3В                   |
| **Датчик уровня 1**       | GPIO2             | XKC‑Y25‑V (цифровой выход)                |
| **Датчик уровня 2**       | GPIO3             | XKC‑Y25‑V (цифровой выход)                |
| **Питание L298N**         | +12V, GND         | Внешний источник                          |
---------------------------------------------------------------------------------------------

## 🛠️ Требования и зависимости

- **ESP‑IDF v6.1** (экспериментальная, но стабильная для этого проекта).  
- Компоненты IDF, автоматически устанавливаемые через `idf_component.yml`:
  - `espressif/onewire_bus`
  - `espressif/ds18b20`
  - `espressif/cjson`
  - `spiffs` (встроен в IDF)
- **Язык**: C (стандарт GNU23).

## ⚙️ Настройка проекта

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/Raven078/arregation_system_tomato.git
   cd arregation_system_tomato
Запустите конфигуратор:

bash
idf.py menuconfig
Обязательные параметры:

WiFi Configuration → WiFi SSID, WiFi Password, TCP Server IP, TCP Server Port.

Dry voltage (mV) for HW-390 и Wet voltage (mV) for HW-390 – подберите под свой датчик (по умолчанию 2500 и 1000 мВ, но могут потребоваться другие значения).

Moisture threshold for pump activation (%) – порог влажности для включения насоса.

Sensor read interval (seconds) – интервал опроса датчиков (сек).

Log send interval (minutes) – периодичность отправки лога (мин).

Serial Flasher Config → Flash size → установите 4 MB (рекомендуется).

Убедитесь, что в корне проекта есть файл partitions.csv (пример):

csv
# Name,   Type, SubType, Offset,   Size, Flags
nvs,      data, nvs,     0x9000,   0x5000,
phy_init, data, phy,     0xe000,   0x1000,
factory,  app,  factory, 0x10000,  2M,
spiffs,   data, spiffs,  ,        1M,
В menuconfig → Partition Table → выберите Custom partition table CSV и укажите partitions.csv.

🚀 Сборка и прошивка
bash
idf.py build
idf.py -p /dev/ttyUSB0 flash monitor   # замените порт на свой
После загрузки устройство подключится к WiFi, синхронизирует время, создаст файл лога и начнёт отправлять данные. В мониторе вы увидите:

Минутные усреднённые JSON (Minute averaged JSON: ...)

Мгновенные JSON при событиях (Instant JSON (event): ...)

10‑минутные записи в файл (10-min record added: ...)

Отправку файла с содержимым (Sent X lines from ..., затем содержимое файла)

📤 Форматы данных
Минутный JSON (отправляется в HH:MM:00)
json
{
  "device": {
    "name": "Теплица помидорник",
    "id": "CC8DA2BFFECC",
    "type": "ESP32-C3_Sensor_Monitor"
  },
  "time": "14:35:00",
  "sensors": {
    "moisture": {"percent": 68.5, "raw": 2048, "voltage_mv": 1650, "unit": "%"},
    "temperature": {"value": 23.45, "unit": "°C"},
    "levels": {
      "sensor_1": {"detected": true, "status": "detected"},
      "sensor_2": {"detected": false, "status": "not_detected"}
    }
  },
  "pump": "off",
  "valve": "off"
}
Файл лога /spiffs/DD_MM_YYYY.txt
10‑минутная запись:

text
2026-05-29 14:20:00;68.5;23.45;1;0
Событие:

text
2026-05-29 14:25:30;EVENT;pump;on
2026-05-29 14:26:45;EVENT;pump;off
2026-05-29 14:26:55;EVENT;valve;on
2026-05-29 14:28:10;EVENT;valve;off
🔧 Отладка и мониторинг
Вывод содержимого текущего файла лога в монитор можно вызвать функцией file_logger_print_file() (она уже вызывается после каждой отправки).

Для ручного вызова добавьте в коде, например, таймер или команду по UART.

В логах присутствуют диагностические сообщения:

10-min timer restarted, next record in X seconds

No readings accumulated for 10-min record (если накоплений не было)

Time not synced yet (если время ещё не синхронизировано)

🧪 Проверка работоспособности
После успешной сборки и запуска:

Устройство получит IP‑адрес.

Синхронизирует время с сервером.

Начнёт отправлять минутные JSON.

При снижении влажности ниже порога и наличии воды на уровне 1 включится насос (на полную мощность).

При пропадании уровня 1 насос выключится, через 10 секунд откроется клапан.

При срабатывании уровня 2 клапан закроется.

Каждые 10 минут в файл будет добавляться усреднённая запись, и файл будет отправляться на сервер.

При наступлении нового дня старый файл отправится, удалится, начнётся новый файл.
